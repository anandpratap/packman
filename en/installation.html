Get PACKMAN

PACKMAN is version controlled by GIT, and hosted at GitHub. There are three ways to get PACKMAN, and the most recommended one is through GIT to get the latest version of PACKMAN. Firstly, change to a directory where you want PACKMAN to reside, and type the following command: 

$ git clone https://github.com/dongli/packman
Cloning into 'packman'...
remote: Counting objects: 2594, done.
remote: Compressing objects: 100% (44/44), done.
remote: Total 2594 (delta 14), reused 0 (delta 0)
Receiving objects: 100% (2594/2594), 460.58 KiB | 76.00 KiB/s, done.
Resolving deltas: 100% (1859/1859), done.
Checking connectivity... done.
After that, you will get a packman subdirectory: 
$ ls packman
README.md bashrc doc framework packages packman setup.sh
Secondly, you could execute setup.sh to check if the version of system Ruby meets the demands: 
$ packman/setup.sh
If everything is ok, the following message will be printed to tell that your ~/.bashrc has been modified: 
[Notice]: Append "source <path_to_packman>/bashrc" into ~/.bashrc. Reopen or relogin to the terminal please.
Otherwise, a new version Ruby will be installed automatically. Please be patient during this process. Then you can relogin the terminal to use PACKMAN.

Configure PACKMAN

To address varied needs, PACKMAN offers multiple configurations. When you first install PACKMAN, run the following command first: 

$ packman config
[Notice]: /Users/dongli/Works/packman/packman.config is generated. Please revise the following settings:
package_root     = ~/.packman/packages
install_root     = ~/.packman
download_command = curl
C compiler       = gcc
C++ compiler     = g++
Fortran compiler = gfortran
[Press ANY key]
which shows some default values. Press ENTER key to proceed and edit the following content: 
package_root = "~/.packman/packages"
install_root = "~/.packman"
download_command = "curl"
use_ftp_mirror = "no"
defaults = {
  "compiler_set_index" => 0,
  "mpi" => "mpich"
}
compiler_set_0 = {
  "c" => "gcc",
  "c++" => "g++",
  "fortran" => "gfortran"
}
To avoid potential conflicts, please do not set package_root and install_root into PACKMAN's directory.
Keyword Meaning
package_root directory to store downloaded packages
install_root directory to store installed packages (better not changed after first set)
download_command can be wget or curl
use_ftp_mirror use FTP mirror, if not set "no", if yes fill "ftp://<IP or domain name>:<port>
defaults compiler_set_index default compiler set index (start at 0)
defaults mpi default MPI library (mpich or openmpi)
compiler_set_* c C compiler for the *-th compiler set
compiler_set_* c++ C++ compiler for the *-th compiler set
compiler_set_* fortran Fortran compiler for the *-th compiler set
